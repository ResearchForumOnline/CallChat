# Q Call Secure Comms License

Q Call is the CallChat secure-communications license for teams that want a private, self-hosted route into messaging, calls, and protected workflows.

Live offer: https://callchat.org/license/

## Offer

| Plan | Price | Scope |
| --- | ---: | --- |
| Monthly | USD $55/month | Unlimited users on one approved public server IP. |
| Annual | USD $550/year | Unlimited users on one approved public server IP, with two months effectively included versus monthly billing. |

The public repository provides the community self-host kit, landing-page assets, documentation, and safe integration boundaries. The paid Q Call / Shield layer is licensed separately and does not grant permission to republish private Shield/ZMath source, entitlement code, production secrets, or customer data.

## Best-Fit Customers

- Small companies that need secure internal messaging without handing every conversation to a third-party SaaS workspace.
- Agencies, consultants, legal teams, clinics, and financial teams that need private client communications and clear admin control.
- Communities and private networks that want Matrix-compatible chat with a branded CallChat front door.
- Operators who want an AI-assisted support/sales front door but still need secrets, keys, and admin tools kept server-side.
- Teams preparing for the quantum era who want a credible PQC-ready roadmap without impossible "unbreakable" security claims.

## Sales Flow

The live page should move visitors through Know, Like, and Trust before the buy button:

1. Know: explain what Q Call protects, who it is for, and how the one-server license works.
2. Like: show clear pricing, simple setup language, and practical examples such as calls, messages, rooms, and protected workflows.
3. Trust: state the security boundary, avoid exaggerated claims, and tell buyers what stays private.
4. Buy: send qualified buyers to the live license form or buy button.
5. Nurture: capture name, email, organization, domain/server IP, and use case so follow-up can guide setup.

## Safe Claim Language

Use:

- "Secure comms with Q Call"
- "Privacy and protection in the quantum age"
- "Quantum-ready roadmap"
- "PQC-ready roadmap"
- "Self-hosted secure communications"
- "Private Shield/ZMath implementation boundary"
- "Unlimited users on one approved public server IP"

Avoid:

- "Unbreakable encryption"
- "Quantum-proof forever"
- "No one can ever read this"
- "Fully audited" unless a real audit report is public
- "Quantum encryption" unless a specific reviewed mechanism is documented with limitations

## AI Agent Sales Behaviour

CallChat agents should do more than answer generic knowledge questions. They should qualify visitors and move serious buyers toward the license page while staying honest:

- Ask what the buyer needs to protect and how many users or rooms they expect.
- Identify whether the buyer is a company, agency, community, or regulated team.
- Explain the USD $55/month and USD $550/year license clearly.
- Recommend the live license page when the visitor asks about price, security, deployment, or Q Call.
- Never request passwords, private keys, access tokens, recovery phrases, payment card details, or Matrix credentials in chat.
- Escalate security-sensitive questions to a private contact route.

## Public/Private Boundary

Safe to publish here:

- Pricing copy.
- Buyer profile.
- Landing-page copy.
- Safe sales-agent prompts.
- Public Matrix, Synapse, Element, TURN, and OpenZero integration guidance.
- Behaviour-level Shield/Q Call descriptions.

Keep private:

- Shield/ZMath implementation source.
- Entitlement and license-verification secrets.
- Production server credentials.
- Payment provider secrets and webhook signing secrets.
- Customer lead lists, invoices, chats, tickets, and room exports.
